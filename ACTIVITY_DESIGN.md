> 本文件紀錄康軒三下數學第七單元各活動的教材分析、認知引導邏輯，
> 以及互動設計的決策原因，供後續 AI 或人員理解「為什麼這樣做」。

## 活動一：乘除的關係

### 教材目標（課本分析）

活動一圍繞「乘除互逆」這個核心概念，分三個層次：

1. **乘法的意義**：幾個幾，建立「每組數量 × 組數 = 總數」的心智模型
2. **除法的意義**：知道總數和每組數量，求組數；或知道總數和組數，求每組數量
3. **乘除互逆**：同一組三個數（如 4、7、28）能寫出乘法和除法，它們互相依存

---

### 活動一互動網頁設計

#### 1. `mul.html` — 乘法累積 × 除法分組

**認知引導邏輯：**

學生學除法常見困難：「除法為什麼和乘法有關？」  
→ 解法：讓學生**親眼看到同一批點**，先用乘法累積出來，再用除法一組組拿走。

**動畫設計決策：**

| 決策 | 原因 |
|------|------|
| 固定 7×4 點陣，不換成不同大小的格子 | 視覺一致性：學生看到的是「同一批28個點」，切換方向更有說服力 |
| 除法：全部點先亮（綠色），每步拿走一組變暗 | 強調除法是「從總數出發，分組拿走」的過程；如果從暗到亮，學生看不出「分掉」的感覺 |
| 乘法：幽靈點（opacity 0.12）先顯示整個格子 | 讓學生預期「會有幾個」，同時不破壞「逐步累積」的驚喜感 |
| 一次4個（橫排）→ 一次7個（直行），用**同一個格子**換方向 | 直接視覺化 4×7 = 7×4，比抽象說明「交換律」有效得多 |
| 動畫結束後才顯示完整算式 | 讓學生先觀察過程，算式是「總結」而非「答案揭示」 |

**互動流程（設計原則：引導多於測驗）：**
```
[乘法模式]
  → 點「下一排」→ 每次出現一排點 + 累積計數
  → 完成後顯示「4×7=28」+ 切換到除法按鈕

[除法模式]
  → 點「拿走一組」→ 最新一組變暗 + 倒數計數
  → 完成後顯示「28÷4=7」+ 切換到「一次7個」按鈕

[切換方向：一次7個]
  → 同一點陣，改為直行動畫，重複上述流程
  → 完成後顯示發現 banner：4×7 = 7×4 = 28
```

#### 2. `machine.html` — 倍機器

**認知引導邏輯：**

從 mul.html 學到點陣後，學生需要一個**抽象橋樑**，從圖像走向數字運算。  
倍機器是課本活動一的核心情境：機器輸入一個數、輸出 N 倍的數。

**教學橋接功能：**
- **正向（→）**：知道輸入，求輸出 = 乘法
- **反向（←）**：知道輸出，求輸入 = 除法
- 同一個機器、同一個倍數，方向不同代表乘除互為逆運算

**UI 設計決策：**

| 決策 | 原因 |
|------|------|
| 機器本體用漸層紫藍色，左右分兩欄 | 視覺上強調「輸入→機器→輸出」的因果流向 |
| 第0列固定顯示範例（兩側都有數字） | 讓學生先「看懂機器怎麼運作」，再開始作答 |
| 箭頭顏色：正向黃色、反向粉色、完成綠色 | 顏色編碼方向性，學生不需要讀文字就能判斷這題要求什麼 |
| 倍數選擇器 ×2–×9，預設 ×4 | ×4 呼應 mul.html 的 4×7=28，形成課程前後呼應 |
| 答錯搖動動畫，不扣分、可重試 | 降低焦慮，保持探索氣氛 |
| numpad 在右側（寬螢幕）| 手機直覺：右手輸入，左眼看機器，視線不需要大幅移動 |

**題目設計原則：**
- 每組 4 題：1 範例 + 1 正向（求輸出）+ 2 反向（求輸入）
- 倒序排列（反向題在後）讓學生先熟悉正向再挑戰反向
- ×4 的範例用 7×4=28，直接引用 mul.html 的數字，強化跨活動連結

---

### 活動一整體認知序列

```
mul.html（圖像層）
    ↓ 「點陣 → 算式」的過渡
machine.html（符號層）
    ↓ 「操作機器 → 發現規律」
inverse.html（規律層）
    ↓ 「規律 → 應用」
unknown.html（應用層）
```

每一步都在前一步的基礎上「去掉一層支架」：
- mul.html 有點陣圖
- machine.html 有機器視覺，但沒有點
- inverse.html 有點陣，但學生要自己發現
- unknown.html 只有純算式，學生靠自己的概念解題

## 設計通則（給未來開發者）

1. **動畫是主角，文字是輔助**  
   學生不看說明，但會跟著動畫走。動畫本身要能傳達概念，文字只是補充。

2. **每次只出現一個新資訊**  
   mul.html 的「每步只加一排/拿走一組」就是這個原則的實踐。

3. **錯誤要溫和，不要懲罰**  
   搖動動畫 + 清空輸入，讓學生重試。沒有計分、沒有機會限制。

4. **跨活動數字要一致**  
   4、7、28 這三個數字在 mul.html、machine.html（×4 範例）、unknown.html（□×4=28）反覆出現，形成認知錨點。

5. **手機優先，寬螢幕增強**  
   基礎版面為單欄手機版；`≥560px` 時切成左右分欄（機器/方程式左，numpad 右）。
