### mul.html — 乘法累積 × 除法分組
**教學目的：** 用點陣讓學生「看見」4×7 的意義，以及換方向觀察得到 7×4（交換律）。

**互動流程：**
1. 乘法：一排排累積（4 個一排，重複 7 次）→ 顯示 4×7=28
2. 除法：28 個全亮，每次拿走一組（4 個變暗）→ 28÷4=7
3. 切換「一次 7 個」：同一個 7×4 點陣，改為直行方向
4. 乘法：一行行累積（7 個一行，重複 4 次）→ 7×4=28
5. 除法：28 個全亮，每次拿走一組（7 個變暗）→ 28÷7=4
6. 兩種都完成後顯示「4×7 = 7×4 = 28」發現 banner

**Canvas 設定：**
- `ROWS=7, COLS=4, TOT=28`，畫布 300×310px
- `groupSize=4`（橫排模式）或 `groupSize=7`（直行模式）
- 點半徑 `DR=14`，間距 `SP=36`，中心 `CX=128, CY=147`
- 幽靈點 opacity 0.12 先顯示全格結構
- 除法：全部點初始為亮綠色（OTC=#2f9e44），每步拿走一組 → lerpColor → 灰色 opacity 0.28
- 顏色常數：`ORC=#e06b00`（每組），`OVC=#7c3aed`（組數），`OTC=#2f9e44`（總數），`GRAY=#aaa`

### machine.html — 倍機器
**教學目的：** 以「機器」隱喻建立乘除正反向，知道輸出找輸入就是除法。

**UI 結構（寬螢幕左右分欄，≥560px）：**
- 左欄：倍數選擇器（×2–×9）+ 機器視覺（輸入欄 | 漸層機器 | 輸出欄）+ 進度
- 右欄：方程式提示框 + 數字鍵盤

**題目資料格式（`DATA` 物件）：**
```javascript
// u: null=範例(兩側都顯示), 'out'=求輸出, 'in'=求輸入
{ i: 7, o: 28, u: null  }  // 範例
{ i: 5, o: null, u: 'out' } // 5×4=?
{ i: null, o: 36, u: 'in' } // ?×4=36
```
- 每個倍數 4 列：第0列範例，第1-3列為題目（共3題）
- 箭頭顏色：fwd=黃，rev=粉，done=綠

**注意事項：**
- `buildMulBar()` 只在初始化時呼叫一次；切換倍數用 `refreshMulBar()`（避免按鈕重複累加 bug）

### inverse.html — 乘除互逆發現
**教學目的：** 同一批點，橫切看除法、直切看另一個除法，找出三個數字的規律。

**互動流程：** 4 輪觀察 → 預測 → 總結  
**狀態機：** `'appear' | 'idle' | 'h_in' | 'h_show' | 'h_out' | 'v_in' | 'v_show' | 'v_out' | 'done'`

---

### unknown.html — 找出 □ 的數
**教學目的：** 知道積和一個因數，用除法找另一個因數（乘除互逆應用）。

**UI 結構：** 與 machine.html 相同（左右分欄，≥560px）
- 左欄：進度點 + 大型方程式顯示 + 答對後顯示除法連結 + 下一題按鈕
- 右欄：方程式提示框 + 數字鍵盤 + 完成 banner

**題目資料格式：**
```javascript
// a=左因數(null=未知), b=右因數(null=未知), p=積
{ a: null, b: 4, p: 28 }  // □ × 4 = 28 → 7
{ a: 3, b: null, p: 27 }  // 3 × □ = 27 → 9
```
- 共 8 題，混合 `□×N=P` 和 `N×□=P` 兩種形式
- 答對後自動顯示對應除法算式（如「28 ÷ 4 = 7」）
- □ 方塊有橘色脈動動畫，答對後變綠色 popIn 動畫
